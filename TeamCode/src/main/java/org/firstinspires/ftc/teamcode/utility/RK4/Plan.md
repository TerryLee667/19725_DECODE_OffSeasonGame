# FTC跑打算法技术文档（考虑车体速度与空气阻力）

## 1. 物理模型

本算法旨在求解炮塔控制参数（水平朝向 \(\phi_{\text{turret}}\) 与仰角 \(\theta\)），使发射的小球在考虑小车自身速度、目标运动、重力及空气阻力的情况下精确命中目标。

### 1.1 坐标系与变量定义

| 符号 | 含义 | 单位 |
| :--- | :--- | :--- |
| \((x_r, y_r, z_r)\) | 小车当前位置 | m |
| \(\vec{v}_r = (v_{rx}, v_{ry}, 0)\) | 小车当前水平速度向量 | m/s |
| \((x_t, y_t, z_t)\) | 目标当前位置 | m |
| \(\vec{v}_t = (v_{tx}, v_{ty})\) | 目标当前水平速度向量 | m/s |
| \(\Delta h\) | 炮口与目标固定高度差 \(z_t - z_r\)（**已知常数**） | m |
| \(v_0\) | 小球相对炮管的发射初速度大小（**待测定常数**） | m/s |
| \(\phi_{\text{turret}}\) | 炮塔相对小车正前方的水平朝向角（**待求解**） | rad |
| \(\theta\) | 炮管仰角，向上为正，\(\theta \le \theta_{\max} < 45^\circ\)（**待求解**） | rad |

### 1.2 小球初始速度合成（相对地面）

炮塔发射动作与小车的自身速度产生叠加，小球相对地面的初始速度向量为：

\[
\begin{cases}
v_{x0} = v_0 \cos\theta \cos\phi_{\text{turret}} + v_{rx} \\
v_{y0} = v_0 \cos\theta \sin\phi_{\text{turret}} + v_{ry} \\
v_{z0} = v_0 \sin\theta
\end{cases}
\]

> **注**：小车速度通常仅在水平面内，不影响垂直初速。

### 1.3 小球运动微分方程（含空气阻力和升力）

考虑到FTC所用小球通常带有通孔，其阻力特性复杂，且会产生升力。采用**阻力和升力模型**以保留调参灵活性：

小球质量为 \(m\)，单位质量受到的空气阻力加速度向量为：

\[
\vec{a}_{\text{drag}} = -\frac{k_x}{m} \cdot |\vec{v}| \cdot \vec{v}
\]

同时，小球会产生竖直向上的升力：

\[
\vec{a}_{\text{lift}} = \frac{k_y}{m} \cdot |\vec{v}| \cdot \hat{z}
\]

其中：
- \(k_x\)：阻力系数（**需实验测定**）
- \(k_y\)：升力系数（**需实验测定**）
- \(\hat{z}\)：竖直向上的单位向量

结合重力加速度 \(g\)，小球在三维空间的运动微分方程为：

\[
\begin{aligned}
\ddot{x} &= -\frac{k_x}{m} \cdot |\vec{v}| \cdot \dot{x} \\
\ddot{y} &= -\frac{k_x}{m} \cdot |\vec{v}| \cdot \dot{y} \\
\ddot{z} &= -\frac{k_x}{m} \cdot |\vec{v}| \cdot \dot{z} + \frac{k_y}{m} \cdot |\vec{v}| - g
\end{aligned}
\]

### 1.4 目标运动模型

假设目标在短时间飞行过程中做匀速直线运动（FTC比赛中目标速度变化较慢）。则预测在时刻 \(t\) 的目标位置为：

\[
\begin{cases}
x_{\text{target}}(t) = x_t + v_{tx} \cdot t \\
y_{\text{target}}(t) = y_t + v_{ty} \cdot t
\end{cases}
\]

### 1.5 拦截条件

存在飞行时间 \(t_f > 0\)，使得小球的水平位置与目标预测位置重合：

\[
\begin{cases}
x_p(t_f) = x_{\text{target}}(t_f) \\
y_p(t_f) = y_{\text{target}}(t_f)
\end{cases}
\]

同时，垂直方向需满足在 \(t_f\) 时刻小球高度恰好达到目标高度（由固定高度差 \(\Delta h\) 决定）：

\[
z_p(t_f) = z_r + \Delta h
\]

## 2. 算法流程

整体算法采用**双层迭代**结构：外层迭代修正炮塔水平朝向，内层通过二分法搜索仰角以匹配目标距离。

### 2.1 主要步骤

1. **输入数据获取**  
   - 目标相对位置向量 \(\vec{r}_{\text{rel}} = (x_t - x_r,\ y_t - y_r)\)  
   - 目标速度向量 \(\vec{v}_t\)  
   - 小车速度向量 \(\vec{v}_r\)  
   - 固定参数：\(v_0\), \(k_x\), \(k_y\), \(m\), \(\Delta h\), \(\theta_{\max}\)

2. **初始估计炮塔朝向**  
   \[
   \phi_{\text{turret}}^{(0)} = \text{atan2}(y_{\text{rel}},\ x_{\text{rel}})
   \]

3. **外层迭代（修正水平瞄准点）**  
   设置最大迭代次数 \(N_{\text{outer}}\)（通常 3~5 次足够）和角度收敛容差 \(\epsilon_\phi\)（如 0.5°）。  
   对于第 \(i\) 次迭代：
   
   - **内层二分搜索求仰角 \(\theta^{(i)}\)**  
     以当前候选 \(\phi_{\text{turret}}^{(i-1)}\) 为固定朝向，在 \([0, \theta_{\max}]\) 范围内二分搜索仰角 \(\theta\)，使得满足如下**等效水平射程匹配条件**：
     
     使用**四阶龙格-库塔法（RK4）** 对运动方程进行数值积分，求解飞行轨迹，直到小球高度达到 \(z_r + \Delta h\)（即命中高度），记录飞行时间 \(t_f\) 与水平位移 \((x_p, y_p)\)。  
     计算预测目标位置 \((x_{\text{pred}}, y_{\text{pred}}) = (x_t + v_{tx} t_f,\ y_t + v_{ty} t_f)\)。  
     定义水平偏差距离：
     \[
     D_{\text{error}}(\theta) = \sqrt{(x_p - x_{\text{pred}})^2 + (y_p - y_{\text{pred}})^2}
     \]
     二分搜索目标是找到 \(\theta\) 使 \(D_{\text{error}}\) 最小（即射程恰好达到预测点）。

   - **更新拦截时间 \(t_f^{(i)}\)**  
     根据内层求得的最佳 \(\theta^{(i)}\) 再次仿真，得到准确的 \(t_f^{(i)}\)。

   - **计算新的预测目标位置**  
     \[
     \begin{aligned}
     x_{\text{pred}}^{(i)} &= x_t + v_{tx} \cdot t_f^{(i)} \\
     y_{\text{pred}}^{(i)} &= y_t + v_{ty} \cdot t_f^{(i)}
     \end{aligned}
     \]

   - **更新炮塔水平朝向 \(\phi_{\text{turret}}^{(i)}\)**  
     根据合成速度原理，需要使小球相对地面的水平初速度方向指向预测拦截点。即解方程：
     \[
     \tan(\phi_{\text{ground}}) = \frac{v_0 \cos\theta^{(i)} \sin\phi_{\text{turret}} + v_{ry}}{v_0 \cos\theta^{(i)} \cos\phi_{\text{turret}} + v_{rx}} = \frac{y_{\text{pred}}^{(i)} - y_r}{x_{\text{pred}}^{(i)} - x_r}
     \]
     其中 \(\phi_{\text{ground}}\) 是所需的地面速度方向角。该方程可通过小角度近似或数值迭代（如牛顿法）求解出新的 \(\phi_{\text{turret}}^{(i)}\)。  
     实用简化方法：利用一阶近似直接计算修正量：
     \[
     \phi_{\text{turret}}^{(i)} \approx \phi_{\text{ground}} - \frac{v_{ry} \cos\phi_{\text{ground}} - v_{rx} \sin\phi_{\text{ground}}}{v_0 \cos\theta^{(i)}}
     \]

   - **收敛检查**  
     若 \(|\phi_{\text{turret}}^{(i)} - \phi_{\text{turret}}^{(i-1)}| < \epsilon_\phi\)，则退出循环。

4. **输出结果**  
   返回最终求得的炮塔水平朝向 \(\phi_{\text{turret}}\) 与仰角 \(\theta\)（角度制或弧度制，视控制需要）。

### 2.2 数值积分方法（RK4）

对于运动微分方程组，采用固定步长 \(\Delta t\)（如 0.01 s）的四阶龙格-库塔法进行积分。  
积分终止条件：小球高度 \(z\) 首次达到目标高度 \(z_r + \Delta h\)（由于仰角小于45°，轨迹单调上升段即可命中）。若轨迹下降阶段才达到该高度，则视为未命中，需调整搜索范围。

### 2.3 二分搜索细节

- **初始区间**：\([0, \theta_{\max}]\)  
- **目标函数**：\(f(\theta) = \text{仿真落点水平距离} - \text{预测目标距离}\)  
  注意：由于合成速度的存在，水平射程与 \(\theta\) 的关系可能非单调，但实验表明在 \(0\sim45^\circ\) 范围内通常是单调递增的。若发现非单调，可改用黄金分割搜索或遍历法（精度要求不高时可行）。

## 4. 实验调参方法

建议按照以下顺序分步测定参数，以确保模型精度。

### 4.1 初速度 \(v_0\) 的测定（水平平射法）

**目的**：在阻力和升力影响下反推真实的炮口初速。

**步骤**：
1. 将小车静置于平整地面，炮塔水平固定（\(\theta = 0^\circ\)）。
2. 精确测量炮口离地高度 \(H\)。
3. 发射小球，用卷尺测量首次落地点到炮口垂足的水平距离 \(R\)。重复 5~10 次，取平均值 \(\bar{R}\)。
4. 预设初始的 \(k_x\) 与 \(k_y\)（例如 \(k_x=0.001, k_y=0.1\)）。
5. 利用 RK4 仿真程序，以 \(\theta=0\) 进行数值积分，调整输入 \(v_0\)，使得仿真落点距离与实测 \(\bar{R}\) 一致。可采用二分法自动搜索 \(v_0\)。
6. 记录此 \(v_0\) 作为初始参考值。

### 4.2 阻力和升力参数 \(k_x\) 与 \(k_y\) 的测定（射表拟合法）

**目的**：通过不同距离的实际仰角数据拟合阻力和升力模型。

**步骤**：
1. **场地准备**：在 1.5m、2.0m、2.5m、3.0m、3.5m 处设置靶标（根据实际射程调整）。
2. **数据采集**：
   - 小车静置，使用步骤 4.1 测得的 \(v_0\)。
   - 对每个目标距离 \(D_i\)，手动调整炮塔仰角 \(\theta_i\)，直到连续命中靶标中心（可多次射击确认）。记录数据对 \((D_i, \theta_i)\)。
   - 建议采集至少 5~8 个不同距离的数据点。
3. **参数拟合**：
   - 固定 \(m\) 和 \(g\)。
   - 构建损失函数：\(L(k_x, k_y) = \sum_{i} \left( D_{\text{sim}}(\theta_i, k_x, k_y, v_0) - D_i \right)^2\)  
     其中 \(D_{\text{sim}}\) 是通过 RK4 仿真得到的射程。
   - 使用网格搜索或单纯形优化算法寻找使 \(L\) 最小的 \(k_x\) 与 \(k_y\)。
   - 搜索范围：\(k_x\) 为 0~0.1，\(k_y\) 为 0~0.5。

### 4.3 动态验证与微调

完成静态标定后，在跑动状态下测试算法命中率。若存在系统性偏差（如总是射近或射偏），可在比赛中通过 Driver Station 实时微调参数：

- **射程整体偏近**：适当增大 \(v_0\) 或减小 \(k_x\)，或增大 \(k_y\)。
- **射程整体偏远**：适当减小 \(v_0\) 或增大 \(k_x\)，或减小 \(k_y\)。
- **侧向偏差**：检查小车速度测量是否准确，或对炮塔朝向增加一个固定偏置补偿。

### 4.4 参数固化

将最终测定的一组参数（\(v_0, k_x, k_y\)）写入机器人配置文件，作为自动阶段的默认参数。在比赛现场可预留一个简单的校准 OpMode，用于快速验证和微调。

## 5. 算力与实时性说明

- **RK4 积分步数**：典型飞行时间约 0.4~0.8 秒，步长 0.01 秒时需计算 40~80 步。每步计算 4 次加速度向量。
- **二分搜索次数**：内层约 10~12 次。
- **外层迭代次数**：通常 3~5 次收敛。
- **总运算量**：约 \(5 \times 12 \times 60 = 3600\) 次基本向量运算，FTC Control Hub（高通骁龙系列）可在 **1~2 毫秒**内完成，完全满足实时控制需求（推荐 20~50Hz 更新频率）。

## 6. 注意事项

1. **传感器数据滤波**：目标位置与速度通常来自视觉处理（如 AprilTag），需进行低通滤波以减少抖动。
2. **发射延迟补偿**：从算法计算出角度到实际发射存在机械延迟，可根据小车速度对目标位置进行前向预测补偿（例如预测 50ms 后的目标位置再输入算法）。
3. **安全边界**：若算法求得的仰角超出物理限制（\(\theta > \theta_{\max}\)），说明目标超出射程，应放弃发射或驱车靠近。
4. **模型优化**：当前采用线性阻力模型和升力模型，计算效率高，适合实时控制。

### 核心修改：从“仰角二分”变为“初速度二分”

原算法结构为：
- 外层迭代调整水平朝向 \(\phi\)；
- 内层对仰角 \(\theta\) 进行二分搜索，找到能命中目标水平距离的仰角。

新场景下：
- 仰角 \(\theta\) 为已知固定值（如 \(30^\circ\)）；
- 炮塔发射初速度 \(v_0\) 成为待求变量；
- 内层改为对 \(v_0\) 进行二分搜索，在合理范围内（如 \(2\,\text{m/s} \sim 15\,\text{m/s}\)）寻找能使小球落点匹配预测目标位置的速度值。

---

### 算法适配步骤

1. **设定初速度搜索范围**  
   根据炮塔电机的性能（最低启动转速对应的初速 \(v_{\text{min}}\) 和最大安全初速 \(v_{\text{max}}\)）确定二分边界。

2. **外层迭代不变**  
   仍然迭代求解水平朝向 \(\phi\)，每次迭代中调用内层二分法计算对应的 \(v_0\)。

3. **内层二分法目标函数**  
   对于候选 \(v_0\)，调用 RK4 仿真（包含小车速度叠加与空气阻力），得到飞行时间 \(t_f\) 和落点水平位置，计算与预测目标位置的误差。  
   若落点距离小于目标距离，则需增大 \(v_0\)；反之减小 \(v_0\)。

4. **可达性判断**  
   若内层二分收敛到边界值仍无法满足精度要求，说明当前目标在固定仰角下无法命中（可能太近或太远），此时可选择放弃射击或输出最接近的可行速度。

### Java 代码修改示例

原方法签名修改为：

```java
public double findMuzzleVelocity(double fixedTheta, double phiTurret,
                                 double robotVx, double robotVy,
                                 double targetRelX, double targetRelY,
                                 double targetVx, double targetVy,
                                 DragParameters drag) {
    double vMin = 2.0;   // 最小初速度 m/s
    double vMax = 15.0;  // 最大初速度 m/s
    double tolerance = 0.05; // 速度容差 m/s

    while (vMax - vMin > tolerance) {
        double vMid = (vMin + vMax) / 2;
        ProjectileState result = simulateTrajectory(vMid, fixedTheta, phiTurret,
                                                    robotVx, robotVy, drag);
        double tf = result.t;
        double predTargetX = targetRelX + targetVx * tf;
        double predTargetY = targetRelY + targetVy * tf;
        double error = Math.hypot(result.x - predTargetX, result.y - predTargetY);

        // 根据落点相对目标的远近调整 v0
        // 注意：在固定仰角下，初速度越大射程越远（单调递增）
        if (Math.hypot(result.x, result.y) < Math.hypot(predTargetX, predTargetY)) {
            vMin = vMid;
        } else {
            vMax = vMid;
        }
    }
    return (vMin + vMax) / 2;
}
```

外层迭代逻辑保持不变，只需将 `findElevationWithRobotSpeed` 替换为 `findMuzzleVelocity` 并传入固定仰角。

### 算力评估

- **内层二分迭代次数**：速度范围 \(13\,\text{m/s}\)，容差 \(0.05\,\text{m/s}\)，约需 \(\log_2(13/0.05) \approx 8\) 次迭代。
- **外层迭代次数**：通常 \(3\sim 5\) 次收敛。
- **单次 RK4 仿真步数**：飞行时间约 \(0.5\sim 1.0\) 秒，步长 \(0.01\) 秒，约 \(50\sim 100\) 步。

总计算量 **显著低于原仰角搜索版本**（因速度搜索范围更窄，迭代次数更少），**Control Hub 完全可实时运行**。

---

### 注意事项

1. **速度边界安全**：实际控制炮塔电机时，需将计算出的 \(v_0\) 映射为电机功率或转速，并确保不超过硬件极限。
2. **近距离目标**：若目标距离很近，二分法可能收敛至 \(v_{\text{min}}\) 仍射程过远，此时可考虑提前退出并返回最小速度（并接受可能打高）。
3. **空气阻力参数的依赖性**：初速度变化时，空气阻力对弹道的影响程度会改变，因此之前通过实验标定的阻力参数 \(k_x\) 和升力参数 \(k_y\) 在可变初速场景下依然有效。

## 7. 自动模式实现
在AutoSelect.java中实现Select函数，输入double relativeX, double relativeY, double robotVx, double robotVy, initialV0, initialTheta，返回仰角、初速度和旋转角
初始化时，设定kx,ky,初速度范围，仰角范围，最优初速度列表和最优仰角列表
若mode="V" 则为最优初速度模式，选择最优初速度列表中的第一个值作为初速度，解得对应仰角，若仰角超出范围则改用列表的下一个值，依次类推，直到找到一个仰角在范围内，返回初速度和仰角。若全部查找完后都无解，则返回失败。
若mode="Y" 则为最优仰角模式，选择最优仰角列表中的第一个值作为仰角，解得对应初速度，若初速度超出范围则改用列表的下一个值，依次类推，直到找到一个初速度在范围内，返回初速度和仰角。若全部查找完后都无解，则返回失败。
若mode="Pv"，先用最优初速度模式，若失败再用最优仰角模式，若仍失败则返回失败
若mode="Py"，先用最优仰角模式，若失败再用最优初速度模式，若仍失败则返回失败
